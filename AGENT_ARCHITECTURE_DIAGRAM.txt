╔══════════════════════════════════════════════════════════════════════════════╗
║                          HEALTH MONITORING                                   ║
╚══════════════════════════════════════════════════════════════════════════════╝

    Every 10 seconds: Agent sends heartbeat ❤️
                      ↓
           Orchestrator receives heartbeat
                      ↓
              Updates last_heartbeat timestamp

    If no heartbeat for 60 seconds:
                      ↓
           Agent marked as FAILED ❌
                      ↓
         Recovery Process Initiated 🔧
                      ↓
        Old agent stopped, New agent spawned ✨
                      ↓
            System continues normally ✅


╔══════════════════════════════════════════════════════════════════════════════╗
║                           METRICS TRACKED                                    ║
╚══════════════════════════════════════════════════════════════════════════════╝

Per Agent:
  • tasks_completed        : Total successful tasks
  • tasks_failed          : Total failed tasks
  • average_task_duration : Mean execution time
  • error_count           : Total errors
  • uptime               : Time since spawn
  • current_tasks        : Active task count
  • child_agents         : Number of spawned children

Per Orchestrator:
  • total_agents         : All active agents
  • agents_by_type       : Count per agent type
  • utilization          : Per-type utilization %
  • queue_size           : Pending tasks
